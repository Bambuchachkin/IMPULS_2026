# IMPULS_2026

# Traffic Model – генерация и анализ сетевого трафика

Проект состоит из двух частей:
- **C++**: консольное приложение, генерирующее CSV-файл
- **Python**: скрипт для вычисления характеристик

## Системные требования

- **Linux** (Ubuntu 20.04/22.04 или WSL)
- **Компилятор GCC** (с поддержкой C++17)
- **CMake** (версия 3.10 или выше)
- **Python 3** (с pip)
- Установленные библиотеки Python (при ручной установке): `pandas`, `numpy`, `matplotlib`

## Быстрый запуск всех тестов (Bash-скрипт)

После клонирования репозитория выполните:

```bash
cd IMPULS_2026/Task_1\(Traffic_Models\)
chmod +x bash
./bash
```
Скрипт автоматически:

    Создаёт виртуальное окружение python/venv и устанавливает нужные библиотеки.

    Собирает C++ проект в папке build

    Находит все файлы в папке Examples/ (любые имена, без расширения или с расширением, кроме .csv/.png).

    Для каждого входного файла запускает C++ генератор, а затем Python-анализ. Результаты (CSV и PNG) сохраняются рядом с входными файлами.

Если Bash-скрипт не может быть использован, следуйте ручной инструкции ниже.
## Ручной способ сборки

1. Сборка C++ приложения
```
cd IMPULS_2026/Task_1\(Traffic_Models\)/
cd build
cmake ..
make
```

Исполняемый файл traffic_model появится в папке build/.

2. Подготовка Python-окружения
```
cd ../python
python3 -m venv venv
source venv/bin/activate
pip install pandas numpy matplotlib
```

3. Запуск тестов вручную

Для каждого входного файла (например, ../Examples/test_uniform) выполните:

Генерация CSV
```
../build/traffic_model ../Examples/test_uniform
```

Анализ CSV и построение графика
```
python3 Analysis.py ../Examples/test_uniform.csv
```

# Входные параметры
    В папке Examples создайте файл имеющий структуру:

    Первая строка – время симуляции (секунды, ≤ 3600)
    Вторая строка – параметры модели:
    uniform: <интервал_сек> <размер_байт>
    poisson: <среднее количество пакетов в сек (интенсивность)> <средний_размер_байт>

Пример содержимого test_uniform:
```
10
uniform 1 1000
```

Результат: test_uniform.csv (столбцы time,size) и test_uniform.png (гистограммы).

# QAM Modulation – моделирование цифровой модуляции

Вторая задача направлена на исследование помехоустойчивости квадратурной амплитудной модуляции (QPSK, 16-QAM, 64-QAM).
Реализованы классы модулятора, демодулятора и источника гауссовского шума. Программа генерирует случайную битовую последовательность, модулирует её, добавляет шум с заданной дисперсией, демодулирует и вычисляет вероятность ошибки на бит (BER). Строится график зависимости BER от дисперсии шума для трёх порядков модуляции.

## Быстрый запуск (Bash-скрипт)

После того как склонировали репозиторий выполните в терминале:

```
cd IMPULS_2026/Task_2\(QAM\)
chmod +x bash
./bash
```

Скрипт:

    Создаёт виртуальное окружение python/venv и устанавливает pandas, matplotlib.

    Собирает C++ проект (cmake, make) в папке build.

    Выполняет симуляцию для 4-QAM, 16-QAM и 64-QAM, сохраняя CSV-файлы в папку CSV/.

    Запускает Python-скрипт для построения графика ber_vs_variance_all.png в той же папке.

# Ручная сборка и запуск

Сборка C++ приложения

```
cd IMPULS_2026/Task_2\(QAM\)
mkdir -p build && cd build
cmake ..
make
```
Исполняемый файл: build/QAM.

Подготовка Python-окружения

```
cd ../python
python3 -m venv venv
source venv/bin/activate
pip install pandas matplotlib
```

Запуск моделирования (из папки Task_2(QAM))

```
./build/QAM 4     # QPSK
./build/QAM 16    # 16-QAM
./build/QAM 64    # 64-QAM
```

Каждый запуск создаёт CSV-файл CSV/ber_variance_<degree>.csv.

Построение графика

```
cd python
python3 QAM_Analysis.py
```

Результат: ../CSV/ber_vs_variance_all.png – график BER для всех трёх видов модуляции.

# Входные параметры

Часть программы, написанная на C++ принимает один аргумент – порядок модуляции (4, 16 или 64).
В коде задаются диапазон дисперсий шума (логарифмическая шкала) и количество передаваемых бит.
Нормировка созвездия обеспечивает единичную среднюю энергию символа.
